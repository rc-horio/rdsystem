# RDシステム エラーメッセージ一覧

## ユーザーが不安になりやすいポイント（改善候補）

1. **技術用語が多い**
   - 「F12」「コンソール」「S3」「CORS」「UUID」「index.json」など
   - → 一般ユーザーには「しばらく時間をおいて再度お試しください」などに寄せる

2. **「失敗」「エラー」の連続**
   - → 「うまくいきませんでした」「ご希望の操作ができませんでした」など、やわらかい表現

3. **ユーザー責任の印象**
   - 「〇〇を入力してください」「〇〇を選択してください」は適切
   - 「確認してください」「ご確認ください」は管理者向けの可能性

4. **誤字の可能性**
   - 「この変「SAVE」ボタン」→「この後「SAVE」ボタン」？

5. **API エラーがそのまま表示**
   - DJI NFZ の `body.extra?.msg || body.error || "API error"` がそのまま出る
   - → ユーザー向けに翻訳・要約したメッセージに変換する余地あり
